## 1. 目的

本書は、企画書、要求仕様書、アーキテクチャ定義書に基づき、MVPおよび初期製品版の基本的な機能構成、データ設計、画面設計、処理設計、外部インタフェースを定義する。

---

## 2. システム概要

本システムは、レガシー資産を収集・解析し、構造情報と業務知識を知識グラフとして蓄積し、ユーザーへ可視化・レビュー・影響分析・移行支援機能を提供する。

---

#### 出力

### 4.2 静的解析機能

- Camel
- SQL
- Shell
- 一部C/C++

#### 処理

- 構文解析
- シンボル抽出

- 例外処理抽出

#### 出力

- Program一覧
- Route一覧
- API一覧
- DBアクセス一覧
- 例外一覧

### 4.3 文書解析機能

#### 処理

- 用語抽出
- コード要素との照合

### 4.4 動的解析機能

- 実行経路一覧

- 未使用コード候補

### 4.5 業務知識抽出機能

#### 処理

- 関連コード群をコンテキスト化

- 構造化結果受領

#### 出力

- 業務機能候補
- 業務ルール候補
- 例外ルール候補

### 4.6 レビュー機能

```mermaid
sequenceDiagram
    participant U as レビュー担当者
    participant UI as Web UI
    participant KG as 知識グラフ
    participant AI as LLMエージェント

    U->>UI: 候補一覧を表示
    UI->>KG: 候補と根拠を取得
    KG-->>UI: 候補・証拠・信頼度
    UI-->>U: レビュー対象を表示
    U->>UI: 承認 / 却下 / 修正
    UI->>KG: レビュー結果を保存
    UI->>AI: フィードバックを反映
```

#### 処理

- 候補一覧表示
- 根拠表示
- 承認／却下／修正
- コメント保存
- 履歴保存

#### 出力

- レビュー結果
- 確認済み知識
- 差戻し項目

## 6. API基本設計

### 6.1 主要API一覧

- `POST /api/projects`
- `GET /api/projects`
- `POST /api/projects/{id}/ingest`
- `POST /api/projects/{id}/analyze`
- `GET /api/projects/{id}/graph`
- `GET /api/projects/{id}/rules`
- `POST /api/projects/{id}/rules/{ruleId}/review`
- `GET /api/projects/{id}/impact`
- `GET /api/projects/{id}/mismatch`
- `GET /api/projects/{id}/modernization-plan`

### 6.2 APIレスポンス設計方針

- JSON形式
- すべての推論結果に `confidence` と `evidenceIds` を持たせる
- レビュー可能データには `reviewStatus` を含める
- 非同期処理には `jobId` を返す

### 6.3 レスポンス例

```json
{
  "ruleId": "BR-0001",
  "text": "顧客区分が法人で本人確認完了かつ反社チェック問題なしの場合に口座開設可能",
  "confidence": {
    "level": "Likely",
    "score": 0.82
  },
  "evidenceIds": ["EV-120", "EV-233", "EV-451"],
  "reviewStatus": "Pending"
}
```

## 9. 処理方式設計

### 9.1 バッチ／非同期処理

以下は非同期ジョブとする。

- 大量資産取り込み
- 静的解析
- 文書解析
- LLM推論
- 再解析
- レポート生成

### 9.2 再解析方式

- ファイル差分を検知
- 変更部分のみ再解析
- 依存先へ影響波及解析
- 旧結果との差分表示

### 9.3 障害時処理

- ジョブ失敗時は失敗理由を保存
- 再実行を可能にする
- 部分成功時は成功範囲を保持する
