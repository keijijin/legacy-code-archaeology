## 1. 目的

本ガイドは、「レガシーコード考古学」MVP を実際に操作し、解析・レビュー・影響分析・移行支援までを体験するための手順を定義する。

## 4. エンドツーエンド手順

### 4.1 プロジェクト作成

```bash
curl -s -X POST http://localhost:8080/api/projects \
  -H 'Content-Type: application/json' \
  -d '{"name":"legacy-customer-demo","description":"利用ガイド用"}'
```

返却された `projectId` を以後使用する。

### 4.2 資産取込

```bash
curl -s -X POST http://localhost:8080/api/projects/{projectId}/ingest \
  -H 'Content-Type: application/json' \
  -d '{"assetType":"JAVA_SOURCE","sourcePath":"samples/legacy-customer-system/java/CustomerRegistrationService.java"}'

curl -s -X POST http://localhost:8080/api/projects/{projectId}/ingest \
  -H 'Content-Type: application/json' \
  -d '{"assetType":"CAMEL_ROUTE","sourcePath":"samples/legacy-customer-system/camel/customer-registration-route.xml"}'

curl -s -X POST http://localhost:8080/api/projects/{projectId}/ingest \
  -H 'Content-Type: application/json' \
  -d '{"assetType":"SQL_DDL","sourcePath":"samples/legacy-customer-system/sql/customer_schema.sql"}'

curl -s -X POST http://localhost:8080/api/projects/{projectId}/ingest \
  -H 'Content-Type: application/json' \
  -d '{"assetType":"YAML_CONFIG","sourcePath":"samples/legacy-customer-system/config/application.yml"}'
```

### 4.3 解析ジョブ投入

```bash
curl -s -X POST http://localhost:8080/api/projects/{projectId}/analyze \
  -H 'Content-Type: application/json' \
  -d '{"jobType":"STATIC_ANALYSIS","forceFullReanalysis":true}'
```

### 4.4 業務ルール候補抽出

```bash
curl -s -X POST http://localhost:8080/api/projects/{projectId}/ai/extract-rules
```

### 4.5 レビュー

ブラウザで開く:

```text
http://localhost:8080/review/
```

1. Project ID を入力
2. 候補を読み込む
3. 根拠（evidenceIds / sourcePath）を確認
4. 承認 / 却下 / 保留

### 4.6 影響分析

```bash
curl -s -X POST http://localhost:8080/api/projects/{projectId}/impact \
  -H 'Content-Type: application/json' \
  -d '{"targetType":"COLUMN","targetName":"customer_id","maxDepth":3}'
```

### 4.7 OpenShift 移行課題抽出

```bash
curl -s http://localhost:8080/api/projects/{projectId}/openshift-migration-issues
```

### 4.8 モダナイゼーション計画

```bash
curl -s http://localhost:8080/api/projects/{projectId}/modernization-plan
```

---

## 5. UI 画面の使い方

### 5.1 Portal（`/`）

- プロジェクト作成
- 移行課題・近代化計画の取得
- Review / Impact UI への導線

### 5.2 Review UI（`/review/`）

- 業務ルール候補の確認
- 根拠表示
- 承認 / 却下 / 保留

### 5.3 Impact UI（`/impact/`）

- COLUMN / ROUTE / PROGRAM 等の影響探索
- 関連テスト表示

---

## 6. 成功判定の観点

- 業務ルール候補が `Pending` で作成される
- レビューで状態が `Approved` / `Rejected` に遷移する
- 影響分析で `impactedNodes` が返る
- OpenShift 移行課題が抽出される
- モダナイゼーション候補が返る

---

## 7. トラブルシュート

- Health が DOWN → DB/Neo4j の起動確認
- 候補が 0 件 → Evidence が不足している可能性
- レビュー UI が 500 → 静的リソースの配置を確認
