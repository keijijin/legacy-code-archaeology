# ADR-2026-003 技術スタックを確定する

- ステータス：Accepted
- 決定日：2026-07-18

---

## 1. 背景

Phase 0 の開始にあたり、実装言語・フレームワーク・DB・ビルドツール・テストツールを確定する必要がある。

---

## 2. 決定事項

### バックエンド
- 言語：Java 21
- フレームワーク：Spring Boot 3.x
- ビルドツール：Gradle（Kotlin DSL）
- API：REST（Spring Web MVC）
- 非同期ジョブ：Spring Batch / Spring Scheduler（MVP段階）

### データストア
- RDB：PostgreSQL 16
- 知識グラフ：Neo4j 5.x
- Vector DB：pgvector（MVP段階、将来的に専用Vector DBへ移行）

### 静的解析
- Java Parser：JavaParser
- Camel Route：Apache Camel Core + XMLパーサ
- SQL DDL：JSQLParser
- YAML / Properties：SnakeYAML / Java Properties API

### LLM接続
- Spring AI（OpenAI / オンプレLLM 切替可能）

### 認証
- Keycloak（OpenShift SSO連携）

### コンテナ・実行基盤
- OpenShift 4.x
- Podman（ローカル開発）

### CI
- GitHub Actions

### テスト
- JUnit 5
- Testcontainers（PostgreSQL / Neo4j）
- Mockito

---

## 3. 採用理由

- Java/Spring はチームの既存スキルと親和性が高い
- JavaParser / JSQLParser は成熟した静的解析ライブラリ
- Spring AI はLLM切替を抽象化できる
- Neo4j はグラフ探索とSpring Data Neo4jで統合しやすい
- OpenShift は標準実行基盤として既に採用決定済み

---

## 4. 影響

- Gradle Kotlin DSL で統一ビルド
- Spring Boot 3 以上（Java 21 必須）
- Testcontainers でDB依存テストを実装
- OpenShift manifest は deploy/openshift/ に集約
