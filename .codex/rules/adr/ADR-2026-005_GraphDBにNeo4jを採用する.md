# ADR-2026-005 Graph DB に Neo4j を採用する

- ステータス：Accepted
- 決定日：2026-07-18

---

## 1. 背景

知識グラフの実装にあたり、Neo4j と Jena/Fuseki を比較する必要があった。

---

## 2. 決定事項

**Neo4j 5.x を採用する。**

---

## 3. 比較

| 観点 | Neo4j | Jena/Fuseki |
|---|---|---|
| 開発生産性 | Spring Data Neo4j が充実 | Java APIは強力だが学習コスト高 |
| クエリ | Cypher が可読 | SPARQL |
| 影響分析 | 多段パス探索が容易 | 可能だが記述が冗長になりやすい |
| RDF厳密性 | 低〜中 | 高 |
| 運用 | 単体運用しやすい | トリプルストア運用が必要 |
| MVP適合 | 高い | 過剰になりやすい |

---

## 4. 採用理由

- 影響分析の多段依存探索を短期間で実装できる
- チームの実装速度を優先できる
- OpenShift 上での運用実績・エコシステムが十分

---

## 5. 将来拡張

- セマンティクス厳密性や標準相互運用が必要になった場合、RDFエクスポートまたは Fuseki 連携を再評価する
