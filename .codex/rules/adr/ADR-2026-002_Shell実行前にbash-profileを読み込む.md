# ADR-2026-002 Shell実行前に .bash_profile を読み込む

- ステータス：Accepted
- 決定日：2026-07-18

---

## 1. 背景

本プロジェクトでは、`oc` や各種開発ツールが `~/.bash_profile` で設定された PATH や環境変数に依存している可能性がある。実際に、Shell から `oc` 実行時に `command not found` が発生した。

---

## 2. 決定事項

Shell コマンド実行前には、原則として `source ~/.bash_profile` を実行し、実行環境を初期化する。

---

## 3. 採用理由

- ログインシェル依存の PATH 問題を回避できる
- `oc` などのCLIが見つからない問題を減らせる
- 開発者端末差異を吸収しやすい
- 手順書・スクリプトに一貫性を持たせられる

---

## 4. 影響

- Shell 手順書では `source ~/.bash_profile` を前置する
- 自動化スクリプトでも必要に応じて環境初期化を行う
- コマンド失敗時には、環境初期化漏れの可能性も確認対象とする

---

## 5. リスク

- `.bash_profile` に副作用の強い処理がある場合、予期せぬ影響が出る可能性がある

### 回避策

- `.bash_profile` の内容を見直す
- CLI 用設定を必要最小限に保つ
- 将来的には専用ラッパースクリプトへ移行する
